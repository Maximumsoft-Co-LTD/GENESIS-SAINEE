# Plan: 20 วัน 4 Missions — เป้าคือ "พร้อมเริ่ม coding v11 เดือนหน้า"

### Context
- ทีม: dev 5, PM 1, QA 1

- v10: 20+ microservices
- Deadline driver: **เริ่ม coding v11 เดือนหน้า**
- ทีม level: ใช้ Claude บ้างแล้ว แต่ยังไม่ systematic

### การ reframe เป้าหมาย (สำคัญมาก)

"Production-ready ทุก mission" + 20+ services + 20 วัน = เป็นไปไม่ได้จริงๆ

แต่ **"พร้อม coding v11"** ทำได้ — และนั่นคือเป้าที่แท้จริง:

| Mission | "Production-ready" (ทำไม่ทัน) | "พร้อม coding v11" (ทำได้) |
|---------|-------------------------------|--------------------------|
| M1 | document ครบ 20+ services | รู้ business logic ของ **core services** พอเขียนใหม่ได้ |
| M2 | requirement sign-off ครบ | รู้ว่า must-have ของ v11 คืออะไร |
| M3 | final architecture | ตัดสินใจ stack + structure แล้ว พร้อม sprint 1 |
| M4 | workflow เป็น standard | ทีมใช้ Claude ใน daily work จริง |

### ขั้นแรก: ต้องนิยาม "core services" ก่อน

```
Tier 1 (ต้อง migrate ใน v11 sprint แรก) → M1 ต้องครบ
Tier 2 (migrate ใน sprint ถัดไป)        → M1 draft พอ
Tier 3 (อาจ deprecate หรือ scope ต่อ)   → skip ก่อน
```

**งานแรกสุด (วันที่ 10–11):** PM + Tech Lead นั่งจัด Tier 1/2/3 ก่อน
